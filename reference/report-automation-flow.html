マーケティング自動化

週次レポート自動化ツールの
仕組み

毎週手作業でまとめていた数字報告を、ツールが全部やってくれる。
その仕組みは、たった 4つのパーツ でできている。

起動 → 収集 → 加工 → 配信

ツールが毎週自動でこの4ステップを実行する

1
Trigger

トリガー — いつ・何をきっかけに動くか

GitHub Actions（cron）

コードをGitHubに置いておくだけで、サーバーを持たずに決まった時間にPythonスクリプトを自動実行できる。「毎週月曜の朝9時に起動」のような設定を書くだけでOK。

サーバー不要 無料枠あり（月2,000分） 設定後は完全自動
schedule.yml
# 毎週月曜 9:00 に起動
on:
schedule:
- cron: '0 0 * * 1'
jobs:
report:
runs-on: ubuntu-latest
2
Source

ソース元 — データがある場所

Shopify API
認証：APIキー発行のみ（最も簡単）
  • 売上・注文数
  • セッション数・CV率
  • 注文単価・リピート率
Meta広告 API
認証：Meta開発者登録が必要
  • 広告費・ROAS
  • クリック数・CPC
  • CVR・CPA
Google広告 API
審査あり（2〜3週かかる）
  • 広告費・ROAS
  • クリック数・CPC
  • CVR・CPA

まず着手すべき順番：Shopify（APIキーだけ）→ Meta広告（開発者登録）→ Google広告（審査申請は今すぐ出しておく）

3
Process

処理する場所 — データを加工してレポートにする

Python スクリプト（GitHub Actions上で動く）

すべての処理の司令塔。データ取得から公開まで一気通貫で実行する

1
各APIからデータ取得

Shopify・Meta・Google の3つのAPIを叩いて、今週の指標をまとめて取得する

2
Google Sheets に生データを保存 蓄積

週ごとのデータを積み上げていくことで、月次・四半期の推移が追えるようになる。人間も直接スプレッドシートを見られる

3
異常値チェック アラート

先週比で30%以上の変化があった指標を自動検知。悪化していたらSlack通知に「⚠️ アラート」として強調する

4
Claude API で改善アクション3選を生成 AI

今週の数値をAIに渡すと「〇〇が課題かも？△△を試してみよう！」のようなカジュアルで前向きなアドバイスを自動生成する

5
HTMLレポートを生成 → surge.sh で公開 公開

数値とアクションをHTMLに埋め込み、surge.shに1コマンドでアップロード。URLが自動で発行される（例：abil-report-20260414.surge.sh）

4
Deliver

届ける先 — 完成したレポートを届ける

Slack（Incoming Webhook）

Slack の設定画面でWebhook URLを1つ発行するだけ。あとはPythonからURLにデータを送るだけで、指定チャンネルに自動投稿される。

設定5分で完了 任意のチャンネルに投稿 URLタップで詳細レポートへ
Slack — #weekly-report
ABiL Bot 月曜 09:00
【週次レポート】4/7〜4/10
─────────────
売上：¥375,000（先週比 +10.3%）
広告費：¥100,000 / 広告費率：26.7%
⚠️ Meta CPA が先週比 +38%
─────────────
詳細レポートを見る →
https://abil-report-20260414.surge.sh
